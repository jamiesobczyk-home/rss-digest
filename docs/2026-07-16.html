How bitdrift scaled to 121 million concurrent gRPC connections on Amazon CloudFront for live telemetry sporting events

bitdrift faced a critical scaling failure during live T20 World Cup broadcasts when traffic surged to 121 million concurrent gRPC connections, with Route 53's Weighted routing policy concentrating all CloudFront edge traffic to a single Network Load Balancer despite having multiple origins available. The root cause was Weighted routing returning only one IP per DNS query, causing all edge nodes to resolve to the same origin for the entire TTL window—a "thundering herd" effect that no amount of origin scaling could fix. Switching to Multi-Value Answer routing, which returns up to 8 IPs per query, immediately distributed traffic across all NLBs and eliminated errors entirely, achieving zero server-side outages at 110K+ peak requests per second.

Hacker News · 12:33 AM

Netstrings (1997)

Netstrings encode strings as [length]:[string], (e.g., "12:hello world!,"), declaring size upfront so applications can verify available storage before reading. This prevents buffer overflow vulnerabilities that plagued protocols like CRLF termination, which lacked size declaration and encouraged unsafe fixed-size buffer implementations. Netstrings provide a simple, secure building block for reliable network protocols and support recursive nesting.
