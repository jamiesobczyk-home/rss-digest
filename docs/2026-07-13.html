Quadrupling code performance with a "useless" if

A developer optimizing a domain-specific compressor discovered a performance bottleneck in a loop that traces references through an array, where each iteration depends on the previous one, limiting performance to memory-access latency. Adding a seemingly redundant `if (j != next_j[i][j])` condition exploits CPU branch prediction to speculate that the value rarely changes, converting the loop from latency-bound to throughput-bound and quadrupling performance. The compiler sees the if as useless since the variable is in a register, but the CPU's speculative execution makes it highly effective.

Backtrack-Free Cursive

The author designed a new cursive script to eliminate backtracking—the need to add strokes like dots on i's and crosses on t's—which disrupts writing flow in English cursive. Analyzing Crime and Punishment in both languages, they found English requires backtracking for 51% of words versus only 6.4% in Russian. Their solution, based on SmithHand, uses single-stroke variants for problematic letters: t gets an auxiliary crossing line (like drawing a mirrored 4), and x uses two mirrored c's instead of diagonal strokes.
