Project Valhalla, Explained: How a Decade of Work Arrives in JDK 28

JDK 28 will integrate Project Valhalla, a decade-long effort to allow objects to be written like normal classes but executed as efficiently as primitives, though it arrives as a preview feature disabled by default. The project solves Java's core limitation that objects are reference types scattered across the heap as pointers, creating inefficient memory layouts and requiring costly indirection for each field access. Valhalla aims to enable dense, contiguous memory layouts so data lies side by side, improving performance as modern CPUs rely heavily on cache efficiency.

So You Want to Define a Well-Known URI

Mark Nottingham, author of the Well-Known URI specification, advises that well-known locations work best when clients need to discover something about an entire site efficiently (like robots.txt), but cautions against using them merely for legitimacy or as a workaround to avoid full URLs, as this creates unnecessary rigidity in deployments. Well-known locations should only be registered if the protocol genuinely requires site-wide discovery; otherwise, they solve a problem that doesn't exist and won't deliver the adoption benefits designers may expect.
