Memory Safe Context Switching

Fil-C provides memory-safe implementations of context-switching APIs including setjmp/longjmp and ucontext functions (used for exception handling and coroutines), preventing dangling stack references that could otherwise lead to stack corruption or security vulnerabilities. Misuses of these APIs—such as restoring contexts after function returns or freeing stacks before context switching—are detected by Fil-C and result in panics rather than undefined behavior. This eliminates the cryptic crashes and potential exploits that arise from executing on dangling stacks in unsafe implementations.
